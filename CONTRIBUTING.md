# Contributing to Packetor

Thank you for your interest in contributing to Packetor.

## How to contribute

1. Fork the repo
2. Create a feature branch
3. Follow the coding standards (TypeScript, deterministic functions)
4. Add tests where applicable
5. Submit a pull request

## Code style

- TypeScript only
- No randomness in core engine
- All rule-packs must be versioned
- All governance functions must be deterministic

## Pull requests

- One feature per PR
- Include a clear description
- Reference related issues
